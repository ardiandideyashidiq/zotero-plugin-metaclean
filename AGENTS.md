# Zotero MetaClean — AGENTS.md

## Quick commands

| Command           | What it does                                                          |
| ----------------- | --------------------------------------------------------------------- |
| `pnpm start`      | Build dev mode, launch Zotero, watch `src/` + `addon/` for hot reload |
| `pnpm build`      | `zotero-plugin build && tsc --noEmit` (esbuild + type-check)          |
| `pnpm lint:check` | `prettier --check . && eslint .`                                      |
| `pnpm lint:fix`   | `prettier --write . && eslint . --fix`                                |
| `pnpm test`       | `zotero-plugin test` (requires running Zotero with plugin)            |
| `pnpm release`    | `zotero-plugin release` (bump version, tag, push → CI releases)       |

## Architecture

- **Entry**: `src/index.ts` → esbuild bundles to `.scaffold/build/addon/content/scripts/meta-clean.js`
- **Lifecycle**: `addon/bootstrap.js` → loads bundled JS → `src/hooks.ts` (`onStartup`/`onShutdown`)
- **Plugin instance**: `Zotero.MetaClean`
- **Config source of truth**: `package.json` `config` block (addonName, addonID, addonRef, addonInstance, prefsPrefix) — NOT just `manifest.json`. Placeholders like `__addonRef__` are replaced during build.
- **Static assets**: `addon/` (manifest, bootstrap.js, locale `.ftl` files, prefs.js, content)
- **Generated files** (committed!): `typings/i10n.d.ts`, `typings/prefs.d.ts` — auto-generated by scaffold
- **Global types**: `typings/global.d.ts` declares `ztoolkit`, `addon`, `rootURI`, `__env__`

## Key source modules

| Module             | Role                                                          |
| ------------------ | ------------------------------------------------------------- |
| `caseEngine.ts`    | Title/sentence/upper/lower case transforms, acronym detection |
| `normalizer.ts`    | Per-field-group normalizers with all-caps detection guard     |
| `fieldRegistry.ts` | Maps Zotero field names → groups + default case modes         |
| `itemProcessor.ts` | Iterates fields on items, applies normalizer, writes changes  |
| `menus.ts`         | Right-click + Tools menu via `MenuManager.registerMenu()`     |
| `notifier.ts`      | Auto-processes new/modified items if pref enabled             |
| `undoLog.ts`       | In-memory + disk-persisted undo stack                         |

## Key quirks

- `pnpm build` runs `tsc --noEmit` **after** scaffold build for type checking only; esbuild does the actual compilation.
- `esbuildOptions` target is `firefox128`.
- ESLint disables `@typescript-eslint/no-unused-vars` for `.ts` files.
- `addon/prefs.js` uses **already-prefixed** keys (e.g. `extensions.zotero.meta-clean.enabled`). Scaffold's `prefs.prefix` config does not re-prefix these.
- `.env` is gitignored; copy from `.env.example` and set `ZOTERO_PLUGIN_ZOTERO_BIN_PATH` + `ZOTERO_PLUGIN_PROFILE_PATH`.
- CI runs lint → build → test sequentially (`test` needs `build` output).
- Release is triggered by pushing a `v*` tag.
- Package manager: both `package-lock.json` and `pnpm-lock.yaml` exist. `.npmrc` has `public-hoist-pattern[]=*@types/bluebird*` for pnpm.
- Tests (mocha + chai) require a live Zotero instance — assert `Zotero.MetaClean` is defined.
- `Zotero.MenuManager` is not in zotero-types defaults; cast via `(Zotero as any).MenuManager`.
- Scaffold renames `.flt` → `${addonRef}-*.flt` and prefixes FTL messages with `addonRef-` at build time.
- Menu l10n strings go in `mainWindow.ftl` (loaded via `MozXULElement.insertFTLIfNeeded`), not `addon.ftl`. FTL messages use `.label = Text` attribute syntax for `MenuManager`.
- `normalizePublicationLike` / `PublisherLike` / `PlaceLike` match `normalizeTitleLike` — they all have an all-caps detection guard (`detectAllCaps` → lowercase first) to prevent ALL-CAPS strings from being treated as acronyms.
